# Changelog

## [0.3.0] - 2026-04-22

### 新增
- **图表标题格式化**
  - 自动识别"图 X-X"、"表 X-X"、"Figure X"、"Table X"等图表标题段落
  - 新增 `caption` 角色，默认居中、五号宋体、段前段后各6磅
  - 三套内置模板均配置 caption 样式；自定义模板支持 AI 提取，未指定时使用默认值

- **参考文献格式化**
  - 基于状态机检测"参考文献"章节，自动将该章节下的条目识别为 `reference` 角色
  - 默认样式：五号宋体、两端对齐、1.25倍行距、悬挂缩进2字符（无首行缩进）
  - `apply_paragraph_format` 新增 `hanging_indent` 支持（负首行缩进 + 左缩进补偿）
  - 三套内置模板均配置 reference 样式

- **格式检测报告**
  - 排版完成后返回详细报告，列出每种角色的应用样式（字体、字号、加粗、对齐、行距）
  - 新增 `_build_report()` 辅助函数，自动将 Pt 值反向映射为中文字号名、对齐枚举转中文
  - 前端结果页新增格式报告表格，暗色主题，仅展示有匹配段落的角色

- **AI 模板解析增强**
  - `SYSTEM_PROMPT` 扩展为7个元素（新增 caption、reference），支持 `hanging_indent` 字段
  - `build_template` 增加 caption/reference 构建逻辑，Kimi 未返回时自动填充默认值

## [0.2.0] - 2026-04-22

### 新增
- **自定义模板 — AI 识别排版规范**
  - 用户上传学校/单位的"格式要求.docx"文档，调用 Kimi K2.6 大模型自动提取排版参数
  - 提取内容覆盖：题目、一/二/三级标题、正文的字体、字号、加粗、对齐、行距、段间距、缩进，以及页边距
  - 解析结果以表格形式展示供用户确认，一键应用到排版流程
  - 新增 `template_parser.py` 模块（读取docx文本 → 调用Kimi API → 解析JSON → 构建模板配置）
  - 新增 `config.py` 配置文件和 `.env.example` 模板
  - 新增 `/parse-template` API 端点
- **前端"自定义模板"卡片**（青色主题区分），点击后展开上传区域，支持解析状态反馈和参数预览表格
- 格式化引擎支持固定值行距（如"固定值20磅"），通过 `WD_LINE_SPACING.EXACTLY` 实现

## [0.1.0] - 2026-04-22

### 新增
- **排版引擎核心** (`formatter.py`)
  - 支持三套排版模板：通用论文、国标期刊、简洁商务
  - 自动识别文档结构（标题 / 一二三级标题 / 正文），基于段落样式名 + 正则模式双重检测
  - 字体设置：中文字体（宋体/黑体/楷体/微软雅黑）+ 英文字体（Times New Roman/Arial），通过 `w:eastAsia` 属性正确处理中英文混排
  - 段落格式：字号、加粗、对齐方式、段前段后间距、行距、首行缩进
  - 页面设置：页边距、页码（页脚居中）
  - 表格格式：三线表自动转换（顶线粗、表头底线细、底线粗，清除左右竖线）

- **Web 应用** (`app.py` + `templates/index.html`)
  - Flask 后端，支持 .docx 文件上传 / 格式化 / 下载
  - 暗色调现代 UI，拖拽上传，模板选择，排版结果统计展示
  - 最大文件限制 30MB

### 修复
- **前置页面保护**：新增 `find_content_start()` 智能检测，自动跳过封面、诚信声明、目录等前置页面
  - 通过"目录"标题定位 → 跳过目录条目（点线/省略号/尾页码识别） → 匹配正文标题模式确定内容起点
  - 前置区域的段落、表格、页边距均保持原样不动
  - 解决了封面页和诚信声明页被错误重排的问题
